Codex project notes for ARF1 syllabus
Last updated: 2026-05-21

Context
- This repo is the ARF1 course syllabus.
- The user edits the source files in RStudio and renders to HTML for GitHub Pages.
- We are migrating/improving the syllabus as a Quarto book.
- Current working branch used in this session: codex-quarto-book.

User goals
- Modernize the syllabus using Quarto book.
- Improve structure, navigation, readability, design, and internal links.
- Keep the document detailed. The user wants the syllabus to protect him as professor by documenting policies, rules, procedures, rubrics, and expectations clearly.
- Make the rendered HTML easy to inspect in a browser and eventually on GitHub Pages/mobile.
- Improve content gradually, with review after each substantial pass.

Important user criteria
- Do not delete content at this stage.
- Do not remove old calendars. The user wants historical calendars because they help plan the next semester.
- Old calendars should remain available in the folder, but they do not need to appear in the student-facing book navigation unless requested.
- Prefer conservative restructuring before rewriting or cutting content.
- It is acceptable for the syllabus to be long, but it must be easier to navigate and search.
- Policies should be centralized and easy to find.
- Student-facing sections should link to official policy sections instead of repeating policy text everywhere.
- Any major content rewrite should be reviewed incrementally with the user.

Known technical notes
- Quarto render works with:
  C:\Users\DELL\AppData\Local\Programs\Quarto\bin\quarto.exe render
- Rendering creates _book/index.html.
- Opening _book directly through file:// has limitations:
  - Search needs a web server or GitHub Pages.
  - YouTube embeds may behave differently under file://.
- After render, Quarto may delete old generated single-page outputs such as index.html and index_files. In this session those were restored with git restore to avoid mixing source restructuring with publication cleanup.
- _book is ignored in .gitignore.

Recent UI/navigation state as of 2026-05-21
- The book uses three main left-sidebar groups:
  - Course Foundations
  - Policies, Activities, and Evaluation
  - Schedule and References
- The top-level "Course Syllabus" item in the left sidebar is intentionally hidden because the course title already links back to the home page.
- The left-sidebar groups should behave like an accordion: when one group is opened, the other open groups should close.
- The left-sidebar group label should become bold when the user clicks that group. On page load, the group containing the current page should be bold.
- The bold state should be simple only. Do not use a background color, border, or heavy visual highlight for the active group.
- The in-page section navigation is handled by 15_page-sections.html.
- Non-home pages should show a sticky local navigation bar near the top with breadcrumbs plus a "Sections" dropdown.
- The sticky local nav should remain visible when scrolling or when jumping to a lower section.
- The "Sections" dropdown is generated from h2/h3 headings and selected policy/callout anchors.
- The home page should not show the sticky "Sections" bar.
- The sidebar accordion and sidebar active/bold behavior are handled by 16_sidebar-accordion.html.
- Be careful not to break either 15_page-sections.html or 16_sidebar-accordion.html when changing navigation.
